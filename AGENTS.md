## 코딩 전 필수 로드 `[MUST]`

코드를 한 줄이라도 짜기 전, 아래 파일을 순서대로 읽는다.

```
1. skills/engineering/operating-protocol.md   ← 모든 규칙의 근간
2. skills/engineering/code-architect.md   ← 아키텍처 패턴
3. skills/engineering/edge-case-hunter.md ← 놓치기 쉬운 케이스
4. CLAUDE.md                      ← ForPleasure 홈페이지 오버라이드 규칙
5. {기능명}/계획.md               ← 구현 체크리스트 (있으면)
```

---

## 절대 원칙 `[MUST]` — 어기면 코드 반려

1. **한국어 주석 필수** — 설계 의도와 규칙 태그(`[ENG]`, `[DRY]` 등)를 한국어로 명시
2. **계획.md 순서 준수** — 체크리스트 순서대로만. 임의로 건너뛰거나 순서 변경 금지
3. **중복 개발 금지** — 구현 전 기존 코드 탐색, 재사용 가능하면 재사용
4. **레거시 코드 삭제** — 새 구현이 대체하면 구 코드를 남기지 않는다
5. **아키텍처 독단 금지** — 패키지 추가·구조 변경은 사령관 확인 후 진행

## 데이팅 서비스 특화 주의 `[SEC]`

- 개인정보(이메일/연락처 등) 수집 폼은 입력값 검증 + 서버 검증 필수.
- 과장·허위 보장 카피 금지. 신뢰·안전 메시지 우선.
- 자세한 제약은 `CLAUDE.md` 5번 섹션 참조.
