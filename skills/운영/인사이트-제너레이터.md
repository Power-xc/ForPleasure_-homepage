# 인사이트 제너레이터 스킬 (Insight Generator Skill)

> **담당:** 기획부
> **Tier:** 1 (Gemini Flash)
> **Skill ID:** `insight`

## Purpose

전 사업부의 데이터를 수집·분석하여 매일 아침 "사령관이 결정해야 할 3가지"를 포함한 모닝 리포트를 생성한다.
의사결정 지연을 방지하고 데이터 기반 판단을 강제한다.

## Trigger

- n8n 스케줄러 매일 09:00 AM 자동 실행
- `@insight report` 명령으로 수동 호출
- 이상 징후 감지 시 긴급 알림 (임계치 초과)

## Process

```
1. 데이터 수집 (n8n → Supabase 쿼리 + 외부 API)
   ├─ YouTube Analytics API  // 구글 공식 API로 채널 통계 수집
   ├─ Supabase (SaaS 지표)   // 우리 DB에서 가입자/이탈자 집계
   ├─ n8n 실행 로그          // 자동화 파이프라인이 잘 돌아갔는지 확인
   └─ Token Watchdog         // 어제 AI API 비용이 얼마였는지

2. 이상 징후 탐지
   ├─ 전일 대비 20% 이상 변동 시 alert 생성
   │   // 갑자기 조회수가 반토막 나거나, 비용이 2배로 뛰면 즉시 알림
   ├─ 에러 빈도 급증 감지
   └─ 비용 임계치 접근 감지 (예산 80% 도달 시)

3. KPI 트렌드 분석 (7일 이동평균 기준)
   // 하루하루 등락은 노이즈일 수 있음 — 7일 평균으로 진짜 흐름 파악

4. 의사결정 사항 추출
   ├─ 데이터 기반 결정 필요 사항 3개 선별 (정확히 3개)
   ├─ 옵션별 영향 분석
   └─ AI 추천안 제시 (근거 포함 — "왜 A를 추천하는지" 명시)

5. 이전 액션 아이템 추적 및 상태 업데이트
   // 결정만 하고 잊어버리는 것을 방지 — 매일 아침 상태 확인

6. Slack/Discord 메시지 포맷팅 및 발송
```

## Constraints

- 의사결정 사항은 정확히 3개 (부족 시 억지로 지어내지 말고 `[현재 긴급 결정 사항 없음]` 배열로 반환할 것) `[IPAI]`
  // 너무 많으면 집중이 안 됨 — 3개가 인간의 집중력 한계에 최적화된 수. 단, 의미 없는 결정 사항을 할루시네이션으로 만들어내는 것은 더욱 치명적임.
- health_score 가중 평균: 수익 40%, 비용 20%, 안정성 20%, 성장 20% `[IPAI]`
- critical alert 발생 시 즉시 발송 (09:00 스케줄 무시) `[ENG]`
  // 서비스 장애나 비용 폭발은 기다리면 안 됨
- 보고서 길이: Slack 메시지 기준 2000자 이내 `[PROD]`

## Feedback Loop

- **input:** `token` 비용 리포트, KPI, 실패 작업, 전일 결정사항
- **detect:** 비용 증가, 성과 하락, blocked 작업, 개선 필요한 스킬
- **downstream:** `@task open`, `@task block`
- **action payload:** `{"action":"@task open","source_skill":"insight","artifact":"daily_decision","next_skill":"task"}`

## Hard Gates

| ID | 조건 | FAIL 기준 |
|----|------|-----------|
| G1 | 결정 제한 | 오늘 결정해야 할 항목이 3개를 초과하거나 의미 없이 생성됨 |
| G2 | 이상 징후 | 비용, 장애, 성장 하락 중 critical 항목을 누락 |
| G3 | 이슈화 | 실행이 필요한 결정을 task로 넘기지 않음 |

## Soft Findings

| ID | 조건 | 조치 |
|----|------|------|
| S1 | 추세 신뢰도 | 단기 노이즈와 7일 추세를 구분 |
| S2 | 스킬 개선 | 반복 실패 스킬의 개선 이슈 제안 |
