# Marketing Funnel Skill

> **담당:** Growth
> **Tier:** 1 (Gemini Flash)
> **Skill ID:** `funnel`


---

## 언제 쓰는가

- "왜 방문자는 많은데 구매가 안 될까?"
- "어드민 보니까 장바구니 담고 결제 안 하는 사람이 많아"
- "유저가 어디서 이탈하는지 모르겠어"
- "마케팅 채널 어디에 집중해야 해?"

---

## 실행 프로토콜

`@funnel analyze {제품/상황}` 트리거가 오면 아래 순서로 실행한다.

### Step 1 — 퍼널 단계 매핑 (AIDA × TOFU/MOFU/BOFU)

제품/서비스에 맞게 각 단계를 구체화한다.

```
[TOFU — Attention: 인지]
목표: 우리 존재를 알게 한다
채널 예시: 인스타그램, 검색광고, 바이럴 콘텐츠, 커뮤니티
지표: 노출수(Impression), CPM, CTR, 신규 방문자 수, MAU/DAU

[TOFU→MOFU — Interest: 관심]
목표: 제품/서비스를 더 알아보게 한다
채널 예시: 랜딩페이지, 상품 상세, 블로그, 유튜브
지표: 체류시간, 스크롤 깊이, 페이지뷰, 이탈률(Bounce Rate), CTA 클릭률

[MOFU — Desire: 욕망]
목표: "갖고 싶다"는 감정을 만든다
채널 예시: 리뷰, 커뮤니티 후기, 비교표, 사회적 증명, 한정 재고
지표: 장바구니 담기, 위시리스트, 찜, 비교 기능 사용률

[BOFU — Action: 구매]
목표: 실제 결제가 일어난다
채널 예시: 결제 플로우, 할인쿠폰, 간편결제, 이메일 리타겟팅
지표: 결제 전환율, 장바구니 이탈률, ROAS, ROI, CPA

[Post-Purchase — Retention: 재구매]
목표: 한번 산 사람이 또 산다
채널 예시: 배송 완료 이메일, 재입고 알림, 커뮤니티
지표: 재구매율(NSM), LTV, 리뷰 작성률
```

---

### Step 2 — 병목 진단 체크리스트

각 단계 전환에서 아래 질문으로 병목을 찾는다.

**TOFU (인지) 병목 신호:**
```
- 방문자 수가 절대적으로 적음 (월 1,000명 미만)
- 검색 노출이 없음 (SEO 미비)
- 광고 없이 유입 채널이 없음
→ 원인: 콘텐츠 부재, 광고 예산 없음, SEO 미구축
→ 처방: 바이럴 콘텐츠 (@viral write), 검색광고, 커뮤니티 배포
```

**TOFU→MOFU (관심) 병목 신호:**
```
- 이탈률 80% 이상 (랜딩 즉시 이탈)
- 체류시간 30초 미만
- 스크롤 30% 미만
→ 원인: 첫 화면이 Value Proposition을 못 전달함
→ 처방: 히어로 메시지 강화, 로딩 속도 개선, 훅 카피 교체
```

**MOFU (욕망) 병목 신호:**
```
- 상품 상세 페이지 체류는 있지만 장바구니 담기 없음
- 리뷰/후기 부재
- 가격 대비 신뢰 요소 없음
→ 원인: 사회적 증명 부족, 구매 불안 해소 안 됨
→ 처방: 커뮤니티 배지, 후기, 주의사항 선제 공개, 재고 긴박감
```

**BOFU (구매) 병목 신호:**
```
- 장바구니 이탈률 70% 이상
- 결제 페이지에서 이탈
- 회원가입 요구로 이탈
→ 원인: 결제 플로우 복잡, 비회원 결제 미지원, 보안 불안
→ 처방: 비회원 결제, 간편결제, 결제 단계 최소화, 보안 마크 표시
```

**Retention (재구매) 병목 신호:**
```
- 1회 구매 후 재구매율 10% 미만 (내구재 제외)
- 구매 후 브랜드 접점 없음
→ 원인: Post-purchase 커뮤니케이션 없음
→ 처방: 배송 이메일, 재입고 알림, 커뮤니티, 로열티 혜택
```

## IPAI 제품 적용 가이드

### HAUL 퍼널
```
TOFU:  인스타그램 숏폼 + 키보드 커뮤니티 바이럴
MOFU:  상품 상세 (커뮤니티 배지 + 스펙 테이블 + 주의사항)
BOFU:  비회원 결제 + 토스페이먼츠 간편결제
NSM:   90일 내 재구매율 → 재입고 알림 이메일
```

### Claimit 퍼널
```
TOFU:  "정부 지원금 못 받는 이유" 콘텐츠 바이럴
MOFU:  보조금 조회 → 자격 매칭 → "나도 받을 수 있어?" 욕구
BOFU:  서류 대필 서비스 결제 (토스) + 신청 자동화
NSM:   연간 갱신율 (다음 지원금 시즌에 재이용)
```

## 핵심 공식 (암기)

```
마케팅 고수의 정의:
"고객이 제품/서비스를 구매하기까지의 과정을 설계하고,
병목을 없애는 일" — 감성적 추리 X, 지표 기반 O

퍼널 진단 우선순위:
전환율이 가장 낮은 단계 = 지금 당장 고쳐야 할 곳

B2C 병목 3대 원인:
1. 못 봤다 (TOFU — 인지 없음)
2. 안 믿는다 (MOFU — 신뢰 없음)
3. 불편하다 (BOFU — UX 문제)
```

## Hard Gates

| ID | 조건 | FAIL 기준 |
|----|------|-----------|
| G1 | 병목 단계 | TOFU/MOFU/BOFU 중 어디가 문제인지 특정하지 않음 |
| G2 | 지표 연결 | 병목 판단에 사용할 지표가 없음 |
| G3 | 처방 | 다음 액션이 채널/카피/제품 중 어디인지 불명확 |
