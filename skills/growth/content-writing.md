# Content Writing Bot Skill

> **담당:** Growth
> **Tier:** 1 (Claude Haiku)
> **Skill ID:** `content-write`

---

## 언제 쓰는가

- "전문가처럼 글 써줘"
- "이 문체로 새 글 써줘"
- "SEO 블로그 글 작성해줘"
- "클릭률 높은 제목 뽑아줘"
- "스토리텔링 구조로 써줘"
- `@content write {모드/주제/타겟}` 트리거 호출 시

> **copywriting과의 차이:**
> - `@copy write` → 광고·상세페이지·구매 전환용 카피
> - `@content write` → 블로그·SNS·뉴스레터·유기적 콘텐츠

---

## 실행 프로토콜

요청 유형을 확인 후 해당 Mode 실행.

---

### Mode A — 전문가 페르소나 글쓰기

입력: `분야 / 타깃 독자 / 목적 / 주제`

```
실행 순서:

① 페르소나 설정 선언
   "[분야] 10년 경력 전문가 관점에서 작성합니다."
   → 이 분야의 실전 경험 기반 인사이트만 사용
   → 교과서적 정의 대신 "현장에서 직접 겪은 것" 언어 사용

② 글 작성 원칙
   - 전문 용어: 사용하되 처음 등장 시 괄호 안에 1줄 풀이
   - 어투: 확신 있게 (모호한 표현 "~인 것 같습니다" 금지)
   - 구조: 핵심 주장 → 근거 → 실전 적용 순서

③ 출력
   - 본문 (목적에 맞는 길이)
   - 독자 수준별 조정 포인트 1줄 메모
```

---

### Mode B — 문체 모방 글쓰기

입력: `참고 글 원문 / 새 주제`

```
Step 1 — 문체 분석 (참고 글 기준)
   ① 문장 평균 길이: {단문/중문/장문}
   ② 어투: {격식체/친근체/중립}
   ③ 구조 패턴: {두berliner론/결론부터/문답형}
   ④ 특징적 표현 패턴: {반복 구조, 접속사 사용, 강조 방식}
   ⑤ 감정 온도: {냉정/따뜻/열정적}

Step 2 — 동일 문체로 새 주제 작성
   → 분석한 5가지 특징을 모두 의식적으로 적용
   → 완성 후 자가 체크: "참고 글 작가가 쓴 것처럼 보이는가?"
```

---

### Mode C — SEO 블로그 글

입력: `주제 / 메인 키워드 / 독자 수준(초보|중급|전문가)`

```
출력 구조:

[도입] — 훅킹 오프닝 (100자 내외)
  공식: 공감형 질문 OR 충격 통계 OR 장면 묘사로 시작
  → 스크롤 없이 읽게 만드는 첫 문장

[목차] — H2 기준 4~5개
  키워드 자연 배치 규칙:
  - 제목(H1): 메인 키워드 1회
  - 도입: 메인 키워드 1회 (첫 100자 안에)
  - 각 H2: 관련 키워드 1회씩
  - 본문: 1000자당 메인 키워드 2~3회 (과하면 패널티)

[본문] — 각 섹션 300자 내외
  - 소제목 → 핵심 포인트 1개 → 근거 or 예시 → 소결

[마무리] — 요약 + CTA
  공식: "핵심 3가지 기억하세요" 요약 → 다음 행동 유도

SEO 메타 출력:
  - 메타 설명 (155자 이내, 키워드 포함)
  - 추천 내부 링크 주제 2개
  - 추천 이미지 ALT 텍스트
```

---

### Mode D — 클릭률 높은 제목 10개

입력: `주제 / 플랫폼(블로그|유튜브|SNS)`

```
패턴별 2개씩:

① 숫자형: "N가지 방법" / "N초 만에"
   → 구체성이 신뢰를 만들고 클릭 전 기대치를 설정

② 질문형: "왜 ~일까?" / "~을 모르면?"
   → 정보 갭 이론: 모른다는 불안이 클릭을 유도

③ 결과 중심형: "~했더니 ~됐다"
   → Before/After 공식: 결과가 보이면 방법이 궁금해짐

④ 비교형: "A vs B: 무엇이 더 좋을까?"
   → 결정을 대신 해주는 콘텐츠 → 고관여 키워드에 효과적

⑤ 비밀 공개형: "아무도 말 안 해준 ~"
   → 독점 정보 욕구 자극 → 공유율도 높음

각 제목 뒤에: "[클릭률 높은 이유: 한 줄]"
```

---

### Mode E — 스토리텔링 글쓰기

입력: `주제 / 삽입할 정보(통계, 기능, 근거)`

```
4막 구조:

① 문제 상황 묘사 (독자가 공감하는 장면)
   → 3인칭 또는 1인칭으로 구체적 상황 묘사
   → "언제, 어디서, 어떤 감정"이 드러나야 함

② 전환점 (해결책 또는 인사이트 등장)
   → 갑자기 등장하지 말고, 작은 힌트 → 큰 깨달음 순서

③ 변화 과정 (Before → After)
   → 삽입할 정보(수치/기능)를 이야기 안에 자연스럽게 배치
   → "이때 알게 된 것이..." 형식으로 정보 전달

④ 결론 + 독자 적용 교훈
   → "당신도 할 수 있다"는 감각으로 마무리
   → 행동 가능한 1가지 교훈
```

```

→ 뉴스레터 시리즈화: @content repackage (Mode D)
```

## Hard Gates

| ID | 조건 | FAIL 기준 |
|----|------|-----------|
| G1 | 목적 명확성 | 글의 목표가 정보제공/전환/신뢰 중 무엇인지 불명확 |
| G2 | 독자 정의 | 타깃 독자와 읽는 상황이 반영되지 않음 |
| G3 | 구조 완결성 | 제목, 본문 구조, 결론 또는 CTA가 없음 |

## Soft Findings

| ID | 조건 | 조치 |
|----|------|------|
| S1 | SEO 요소 | 검색 의도, 제목 후보, 메타 설명 권장 |
| S2 | 재활용성 | 리패키징 가능한 핵심 문장 표시 |
