# 마케팅 전략봇 스킬 (Marketing Strategy Skill)

> **담당:** 그로스부
> **Tier:** 2 (Claude Sonnet)
> **Skill ID:** `strategy`


---

## 언제 쓰는가

- "우리 제품 마케팅 전략 짜줘"
- "경쟁사 분석해서 어떻게 포지셔닝해야 해?"
- "타겟 고객층이 누군지 모르겠어"
- "새 제품 런칭 전략이 필요해"
- "가격을 어떻게 책정해야 해?"

---

## 실행 프로토콜

`@strategy analyze {제품/상황}` 트리거가 오면 아래 4단계를 순서대로 실행한다.

---

### Step 1 — 3C 분석 (시장 환경 스캔)

세 주체를 동시에 분석해 전략의 출발점을 잡는다.

```
[Company — 자사]
강점: {우리만 가진 것, 경쟁사가 따라하기 어려운 것}
약점: {지금 당장 고치기 어려운 내부 한계}
핵심 역량: {우리가 가장 잘하는 한 가지}

[Customer — 고객]
페인포인트: {고객이 지금 겪는 가장 큰 불편}
욕구: {고객이 진짜 원하는 것 (행동 뒤 동기)}
구매 결정 기준: {무엇을 비교해서 사는가}
세그먼트 힌트: {나이/직업/상황 등 공통점}

[Competitor — 경쟁사]
직접 경쟁: {같은 카테고리 플레이어}
간접 경쟁: {우리 예산을 빼앗는 대안들}
경쟁사 약점: {고객 불만, 빈 공간(Gap)}
```

**3C 핵심 질문:** "경쟁사가 못 하고, 고객이 원하고, 우리가 잘하는 것은?"
→ 이 교집합이 전략의 핵심이다.

---

### Step 2 — SWOT 분석 (전략 방향 결정)

3C에서 추출한 사실을 4분면에 배치한다.

```
내부 요인:
  Strength  (강점): {경쟁 우위 요인}
  Weakness  (약점): {내부 제약, 자원 부족}

외부 요인:
  Opportunity (기회): {시장 트렌드, 경쟁사 공백}
  Threat      (위협): {규제, 대형 경쟁자, 경기}
```

**전략 매트릭스:**

| | 기회 (O) | 위협 (T) |
|---|---|---|
| **강점 (S)** | SO 전략: 강점으로 기회를 최대화 | ST 전략: 강점으로 위협을 방어 |
| **약점 (W)** | WO 전략: 기회를 통해 약점 보완 | WT 전략: 약점·위협 최소화, 철수 고려 |

→ 가장 실행 가능한 전략 사분면을 **1개 선택**해 집중한다.

---

### Step 3 — STP (타겟 설정 + 포지셔닝)

**Segmentation — 시장 쪼개기:**
```
분류 기준:
  인구통계: 나이, 성별, 직업, 소득
  심리통계: 라이프스타일, 가치관, 관심사
  행동 기준: 구매 빈도, 사용 목적, 충성도
  지역: 국내/해외, 도시/농촌

각 세그먼트 크기와 접근 가능성 평가
```

**Targeting — 어디에 집중할 것인가:**
```
집중 공략 세그먼트 선택 기준:
  1. 시장 규모 (크고 성장하는가?)
  2. 접근 가능성 (우리가 도달할 수 있는가?)
  3. 수익성 (CAC 대비 LTV가 충분한가?)
  4. 경쟁 강도 (이미 과점 시장인가?)
```

**Positioning — 마음속 자리 잡기:**
```
포지셔닝 스테이트먼트 공식:
"{타겟 고객}에게 {우리 제품}은
{경쟁 대안]과 달리 {핵심 차별점}을 제공한다.
왜냐하면 {증거/근거}가 있기 때문이다."

포지셔닝 맵: X축 {차원1} × Y축 {차원2}
→ 우리 위치 vs 경쟁사 위치 시각화
```

---

### Step 4 — 4P/4C 전술 믹스

각 P를 고객 관점의 C로 검증한다.

**Product / Customer Solution:**
```
핵심 제품: {기본 기능}
확장 제품: {차별화 요소, 브랜드, 서비스]
잠재 제품: {미래에 추가할 수 있는 것}
포지셔닝에 맞는 강조 포인트: {무엇을 앞세울 것인가}
```

**Price / Customer Cost:**
```
가격 전략 선택:
  침투 가격: 낮게 시작 → 점유율 확보 → 점진적 인상
  스키밍:    높게 시작 → 얼리어답터 → 점차 인하
  가치 기반: 고객이 인식하는 가치에 맞춤
  경쟁 기반: 경쟁사 ±X% 내에서 설정

심리 가격 기법: 9,900원 / 번들링 / 기준가 설정
```

**Place / Convenience:**
```
유통 채널:
  직접: 자사몰, 앱 (마진 최대화)
  간접: 쿠팡, 무신사, 플리마켓 (트래픽 확보)
  디지털: SNS 쇼핑, 카카오톡 채널

핵심 질문: 고객이 사고 싶을 때 어디서 찾는가?
```

**Promotion / Communication:**
```
채널별 메시지 전략:
  TOFU (인지): 인스타 숏폼, 유튜브 쇼츠, 커뮤니티 바이럴
  MOFU (고려): 블로그, 상세페이지, 유튜브 리뷰
  BOFU (구매): 리타겟팅 광고, 할인쿠폰, 후기
  Retention:  이메일, 카카오 알림톡, 재입고 알림

예산 배분 원칙: BOFU 40% / TOFU 40% / MOFU 20%
(이미 관심 있는 사람과 새 유입에 집중)
```

### Step 5 — 전략 보고서 출력 형식

```markdown
## {제품명} 마케팅 전략 보고서

### 전략 한 줄 요약
"{타겟 고객}의 {페인포인트}를 해결하는 {포지셔닝}으로,
{주력 채널}을 통해 {목표 수치}를 달성한다."

### 3C 핵심 인사이트
- 자사 강점: {1~2가지}
- 고객 핵심 욕구: {1가지}
- 경쟁사 공백: {우리가 들어갈 빈 자리}

### SWOT 선택 전략
→ {SO/ST/WO/WT} 전략 채택
이유: {짧은 근거}

### 타겟 세그먼트
1순위: {인구통계 + 행동 특징 + 페인포인트}
2순위: {선택적 확장 세그먼트}

### 포지셔닝 스테이트먼트
{공식 적용 결과}

### 4P 전술 요약
| P | 핵심 결정 | 이유 |
|---|-----------|------|
| Product | {강조 포인트} | |
| Price | {가격 전략 + 가격대} | |
| Place | {1순위 채널} | |
| Promotion | {1순위 채널 + 메시지 톤} | |

### 지금 당장 할 수 있는 것 (3가지)
1. {이번 주 내 실행}
2. {이번 달 내 실행}
3. {분기 내 실행}

### 검증 지표
| 전략 요소 | 측정 지표 | 목표값 |
|-----------|-----------|--------|
| 포지셔닝 인지 | 브랜드 서베이 / SNS 언급 | - |
| 타겟 도달 | 광고 세그먼트 CTR | 2% 이상 |
| 전환 효율 | ROAS | 3 이상 |
| 재구매 | 90일 재구매율 | 30% 이상 |
```

## IPAI 제품 적용 가이드

### HAUL 전략 요약
```
3C 교집합: 키보드 직구 피로 + 개인통관 장벽 + AI 파이프라인
STP: 20~30대 키보드 커뮤니티 유저, 직구 경험 있음
포지션: "검색 한 번으로 끝나는 키보드 직구"
4P 강조: Place(자사몰 직접) + Product(AI 파이프라인 차별화)
```

### Claimit 전략 요약
```
3C 교집합: 서류 복잡성 + 정보 불균형 + AI 자동화
STP: 소상공인/자영업자/청년, 보조금 인식 있으나 신청 포기 경험
포지션: "신청 안 해서 못 받는 보조금 없게"
4P 강조: Promotion("못 받는 이유" 콘텐츠) + Price(성과 기반 수수료)
```

## 핵심 공식 (암기)

```
마케팅 전략 = 포지셔닝의 일관성
"모든 4P가 같은 포지셔닝을 향해야 한다."
가격이 프리미엄인데 채널이 최저가 플랫폼이면 실패.

STP 결론이 틀리면 4P가 아무리 정교해도 헛수고다.
→ 세그먼트 선택에 시간을 80% 쓰고, 전술은 20%로 집행.

작은 시장에서 1등 > 큰 시장에서 5등
CAC를 낮추는 가장 빠른 방법: 타겟을 좁혀라.
```

## Hard Gates

| ID | 조건 | FAIL 기준 |
|----|------|-----------|
| G1 | 3C 분석 | Company, Customer, Competitor 중 하나라도 누락 |
| G2 | STP/4P 연결 | 분석 결과가 실행 전략으로 이어지지 않음 |
| G3 | 우선순위 | 해야 할 일이 나열만 되고 우선순위가 없음 |

| ID | 조건 | 조치 |
|----|------|------|
| S1 | 실험 설계 | 전략별 검증 실험과 성공 지표 제안 |
| S2 | 콘텐츠 연결 | copy/content/email 후속 스킬 후보 표시 |
