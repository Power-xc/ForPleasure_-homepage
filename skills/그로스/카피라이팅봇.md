# 카피라이팅봇 스킬 (Copywriting Bot Skill)

> **담당:** 그로스부
> **Tier:** 1 (Claude Haiku)
> **Skill ID:** `copy`

---

## 언제 쓰는가

- "상세페이지 카피 써줘"
- "광고 문구 여러 버전 뽑아줘"
- "헤드라인이 약한 것 같아, 다시 써줘"
- "페르소나별로 다른 카피가 필요해"
- `@copy write {제품/타겟/플랫폼}` 트리거 호출 시

> **바이럴-스크립트.md와 차이:** 이 스킬은 구매 전환용 카피(상세페이지·광고·랜딩) 전용.
> 숏폼 대본은 `@viral write`를 쓴다.

---

## 실행 프로토콜

`@copy write {입력}` 트리거가 오면 요청 유형을 판단 후 해당 Step 실행.

---

### Mode A — 상세페이지 카피 (7파트 풀세트)

입력: `제품명 / 가격 / 타겟 / Pain Point / 차별점`

```
① 훅 헤드라인 (3초 안에 잡기)
   공식: [결과] + [기간/조건] + [반전 또는 의외성]
   예시 후보 3개 출력 (A: 숫자형 / B: 질문형 / C: 선언형)

② 공감 문구 (Pain Point)
   "혹시 이런 경험 있으신가요?" 형식으로 3줄 이내
   → 고객이 속으로 "맞아, 나 얘기잖아" 할 수 있어야 함

③ 솔루션 소개 (15자 이내 태그라인 + 2~3문장 설명)

④ 핵심 기능 3가지 + 혜택
   형식: [기능명] → [고객이 얻는 것] (기술 설명 금지, 혜택 언어만)

⑤ 사회적 증거 (후기 예시 3개)
   형식: 직업/상황 + 구체적 수치 + 감정 변화
   예: "2주 만에 3kg 감량. 다이어트 3번 실패했던 저도 됐어요 — 직장인 32세"

⑥ FAQ 3개
   가장 흔한 반론 3개를 질문 형식으로 + 안심시키는 답변

⑦ CTA 3가지 버전
   A: 긴급성 소구 ("오늘만 이 가격")
   B: 혜택 소구 ("지금 시작하면 X를 드려요")
   C: 손실 회피 소구 ("이 기회를 놓치면 다시 없습니다")
```

---

### Mode B — 광고 카피 10세트 (심리 소구별)

입력: `제품 / 플랫폼 / 목표(클릭|구매)`

각 세트 형식: `헤드라인 | 본문 (2문장) | CTA | [공략 심리]`

```
① 공포/손실 소구 × 2  → 손실 회피 편향 (Kahneman)
② 혜택/이득 소구 × 2  → 보상 기대감
③ 사회적 증거 소구 × 2 → 군중 심리 (Cialdini)
④ 호기심 소구 × 2     → 정보 갭 이론 (Loewenstein)
⑤ 긴급성 소구 × 2     → 희소성 원리 + 마감 효과

각 카피 마지막 줄: "[공략 심리: {한 줄}]"
```

---

### Mode C — 페르소나별 카피 (세그먼트 메시지)

입력: `제품 / 페르소나 A, B, C (나이·직업·상황)`

```
페르소나마다 별도 카피 세트 출력:
- 강조 혜택: 이 페르소나가 가장 가치 있게 여기는 것
- 언어 톤: 이 페르소나가 실제로 쓰는 말투 (직장인 vs 주부 vs 대학생)
- 공략 감정: 성취감 / 안도감 / 소속감 / 절약 만족감 중 선택
- 카피 (헤드라인 + 본문 1문장 + CTA)

원칙: 같은 제품, 다른 언어. 기능 설명이 아닌 감정 언어로.
```

---

### Mode D — 훅 공식 10개 (SNS 콘텐츠용)

입력: `주제 / 플랫폼 (인스타|틱톡|유튜브)`

```
① "아무도 말 안 해준 ~" 공식 × 2   → 독점 정보 욕구
② "당신이 ~하는 이유" 공식 × 2     → 자기 이해 욕구
③ "~했더니 ~됐다" 결과 공식 × 2   → 결과 확인 욕구
④ "모두가 틀렸다, 사실은~" × 2    → 반전·도발 욕구
⑤ "X년 만에 알게 된 ~" 후회 공식 × 2 → 후회 회피 욕구

각 훅 뒤에: "효과적인 이유: {한 줄}"
```

## 출력 원칙

- 기술 언어 금지 → 혜택·감정 언어만 (예: "AI 기반" X → "3초면 끝" O)
- 수치는 구체적으로 (예: "빠른" X → "72시간 이내" O)
- 각 모드 완료 후 스킬 체이닝 제안 출력:
  ```
  → 이 카피를 A/B 테스트하려면: @funnel analyze
  → 고객 후기가 있다면: @voc analyze로 카피 강화 가능
  ```

## Hard Gates

| ID | 조건 | FAIL 기준 |
|----|------|-----------|
| G1 | 핵심 CTA 포함 | CTA가 없거나 사용자가 다음 행동을 알 수 없음 |
| G2 | 타겟 페르소나 반영 | 모든 카피가 일반론으로만 작성됨 |
| G3 | 반론 처리 단서 포함 | 가격·필요성·긴급성 중 최소 1개 반론을 다루지 않음 |

## Soft Findings

| ID | 조건 | 조치 |
|----|------|------|
| S1 | A/B 변형 수 | 플랫폼별 3개 이상 변형 권장 |
| S2 | 고객 언어 반영 | VOC 입력이 있으면 실제 표현을 우선 사용 |
